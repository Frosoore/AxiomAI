Sur le projet AxiomAI, exécuter le travail de façon directe et pragmatique.

**Why:** L'utilisateur a déjà fait le gros du travail préparatoire (le doc `maintenance/AXIOM_AI_UPGRADE_DETAILS.md`, très détaillé avec refs `fichier:ligne` et code avant/après). Il ne veut pas de couche de méta-documentation supplémentaire ni de cérémonie de process qui ralentit.

**How to apply:**
- Implémenter / corriger / avancer directement sur le code plutôt que produire de la doc explicative en amont.
- Seule doc autorisée/attendue : le `TODO.md` et le `CHANGELOG.md` de chaque étape (cf [[feedback-maintenance-workflow]]). Le `DOC.md` reste léger.
- Ne PAS utiliser les skills superpowers (brainstorming, TDD, etc.) sur ce projet — l'utilisateur l'a demandé explicitement.
- Garder les réponses et le travail concis et orientés action.
