L'utilisateur d'AxiomAI **ne code pas** : il pilote tout le projet via des agents IA. Il est néophyte
sur les sujets techniques (archi, formats, dépendances, etc.).

**How to apply:**
- Avant de lui demander de trancher une décision technique, **expliquer les tenants et aboutissants en
  langage accessible** (analogies, tableaux de trade-offs difficulté/maintenance/risque), sans jargon non défini.
- Sur un choix d'outil/dépendance/archi, présenter les options concrètement (ce que ça change pour *lui*),
  pas juste un nom. Il valide en connaissance de cause (ex. a choisi `tomlkit` après explication des 3 options).
- Garder le travail rigoureux mais la communication pédagogique. Voir [[feedback-execution-style]].
